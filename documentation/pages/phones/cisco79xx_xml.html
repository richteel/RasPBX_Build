Cisco 79xx XML

Cisco 79xx IP Phones require that a configuration file be located at the root of the TFTP Server referenced in option 150 of the DHCP Sever that provided the IP Address for the phone. The XML file has several sections and options. The phones require that the XML file is in the expected format and contains valid values. It is easy to have one value off and prevent the phone from setting up properly.


Important note regarding required SIP Software

SIP Software is required for Cisco IP Phones to work with FreePBX. Information regarding obtaining, uploading, and configuring the SIP Software and IP Phones is found in the /device/loadInformation section below.


XML Template

The easiest way to get started with correctly creating a configuration file is by copying an existing template and modifying the values to suite your needs. Once you do that, you are able to go further and add or modify other values by reading the Cisco documentation online. One helpful resource is Cisco Unified IP Phone 7900 Series and Cisco IP Communicator XML Object Support

  1. Obtain the MAC Address for your IP Phone. There are several ways to get the MAC Address. It is typically on a sticker/label on the phone and is labeled MAC. It is a series of 12 letters and numbers from 0 to 9 and A to F.

  3. Save the file with the following name, replacing the text <MAC Address> with the phone's MAC Address.
    SEP<MAC Address>.cnf.xml
  4. Replace the following values in the template
    • [RasPBX IP Address]: The IP Address of the RasPBX server
      NOTE: The three URLs may be left blank if the features are not to be used.
      • /device/callManagerGroup/members/member/callManager/name
      • /device/callManagerGroup/members/member/callManager/processNodeName
      • /device/authenticationURL
      • /device/directoryURL
      • /device/servicesURL
    • [Phone Label]: A label to be displayed in the top right of the phone's screen. This may be a person's name, a location, or the extension. This value cannot be longer than 12 characters.
      • /device/sipProfile/phoneLabel
    • [Line Label]: A label to be displayed next to the buttons along the right of the screen. I typically use these for lines and label them "Line 1 - Ext 110" and "Line 2 - Ext 110", where 110 is the extension number
      • /device/sipProfile/sipLines/line button="1"/featureLabel
      • /device/sipProfile/sipLines/line button="2"/featureLabel
    • [User Extension]:The extension number for the phone
      • /device/directoryURL
      • /device/sipProfile/sipLines/line button="1"/name
      • /device/sipProfile/sipLines/line button="1"/displayName
      • /device/sipProfile/sipLines/line button="1"/contact
      • /device/sipProfile/sipLines/line button="1"/authName
      • /device/sipProfile/sipLines/line button="2"/name
      • /device/sipProfile/sipLines/line button="2"/displayName
      • /device/sipProfile/sipLines/line button="2"/contact
      • /device/sipProfile/sipLines/line button="2"/authName
    • [Extension Secret]: The Secret value/password for the extension. It must be numeric and cannot be more than 8 numbers.
      • /device/sipProfile/sipLines/line button="1"/authPassword
      • /device/sipProfile/sipLines/line button="2"/authPassword
  5. Save the file
  6. Copy the file to the TFTP Server using an application such as WinSCP. The TFTP root folder is located at /tftpboot on the Raspberry Pi

    You may use an IP Address for another NTP Server. The one in the template is for time-a-g.nist.gov in Gaithersburg, Maryland
  • /device/loadInformation
    You will need the SIP Software for your phone. Here are some sources for the SIP Software.
    • 7941 & 7961 Phones may find the SIP Software at https://git.synchron.com.ua/sysadmin/synchron_voip/-/tree/master/tftp/Cisco_Firmwares
      I recall seeing someone state that is the best version to use is cmterm-7941_7961-sip.8-5-4.zip. The newer v9 is plagued with issues.
    • SIP Software for 7975 Phones may be downloaded from Cisco but only until 30 June 2023, as that is the end support date for the phone. If you do not have a support contract, you may not download the latest version, but you may be able to download the previous version. The downloads are at https://software.cisco.com/download/home/281346597/type/282074288/release/9.4(2)SR3?i=!pp . You will need an account, but it is free to setup. This is the best option for obtaining the software.
